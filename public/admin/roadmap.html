Bloqueado em config (precisam ser cadastrados na Vercel)

PENDÊNCIA

Afiliados de remessa com link real

Hoje em produção: AFFILIATE_WISE_LINK, REMITLY, WU e KAYAK têm env vars setadas mas o conteúdo é literal https://wise.com/invite/placeholder — alguém colou placeholder e nunca trocou.

How to apply: cadastrar nos programas (Wise Partnerize, Remitly Impact, etc.), pegar o link real e atualizar na Vercel → Settings → Environment Variables. Detalhes em AUDITORIA_PARCERIAS.md.

GA4 + Meta Pixel

Tracking de eventos e retargeting estão prontos pra plugar. Faltam só os IDs.

How to apply: criar property em analytics.google.com e Pixel em business.facebook.com, passar os IDs e a gente seta as env vars VITE_GA4_MEASUREMENT_ID e VITE_META_PIXEL_ID.

Push notifications (VAPID keys)

Todo o código de push (subscribe, send, tópicos) já está pronto em api/push/. Falta só a chave VAPID.

How to apply: rodar npx web-push generate-vapid-keys e setar VAPID_PUBLIC_KEY + VAPID_PRIVATE_KEY na Vercel.

Próximo (pós-lançamento do Bolão)

NEXT

Ponte entre estática e app

Hoje quem entra em /guias/cnh/ não vê o app. Quem está em /app/feed não vê os guias. Cada lado fica isolado.

  • Adicionar header consistente com link "App" nas páginas estáticas
  • Adicionar item "Guias" no menu do app linkando pra /guias/
  • Considerar uma "topbar global" idêntica em ambos os mundos

OG image dinâmico (PNG)

Hoje usamos SVG estático. Alguns scrapers (FB/IG mais antigos) preferem PNG. Soluções: converter manualmente o SVG existente pra PNG, ou adicionar @vercel/og pra gerar imagem dinâmica per-grupo (com nome do grupo, contador de membros, etc).

Tracking de eventos no app

Quando GA4 + Pixel estiverem ativos, instrumentar eventos chave: bolao_grupo_criado, bolao_palpitou, bolao_convite_enviado, remessa_clique, voos_buscou. Permite medir conversão real do funil.

Push: cobrança de palpites

Cron diário que dispara push pra membros que ainda não palpitaram, lembrando do prazo. Tópico bolao já existe no PushPrompt.jsx.

Estratégico (decisão grande, depois da Copa)

LATER

Migrar pra Next.js (ou Astro)

Hoje a stack é 3 paradigmas: Vite SPA (React app), HTML estático (landing/guias) e serverless functions (api/). Funciona, mas duplica configuração e complica navegação entre os mundos.

Next.js consolidaria tudo: SSR para SEO, React Server Components para páginas estáticas, e App Router para o app interativo — uma stack só. Astro é uma alternativa mais leve focada em conteúdo.

Why considerar: à medida que adiciona features, a duplicação de "header/footer/auth/menu" entre estática e app cresce. Em algum momento a manutenção vai pesar.
How to apply: não fazer durante crescimento intenso (Copa). Avaliar em julho/agosto, quando o pico passar.

Auth federada (Google / Apple) no app

Hoje o app usa email + Supabase auth quando o user faz onboarding. Pra reduzir friction, considerar OAuth com Google e Apple — especialmente útil pro Feed e Bolão (onde a barreira de criar conta hoje é alta).

Dashboard de comunidade por cidade

Hoje as comunidades existem mas não têm página dedicada com ranking de membros, posts pinados, eventos próximos. Pode virar a "praça pública" de cada cidade brasileira nos EUA.

App nativo (iOS/Android via Capacitor ou Expo)

O PWA atual já permite "Add to Home Screen". Pra ter ícone na App Store + push iOS confiável, considerar wrapper nativo. Apenas se métricas mostrarem que o PWA tá perdendo retenção vs apps nativos concorrentes.
